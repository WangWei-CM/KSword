# KSword 多语言语言包规范

KSword 主程序从可执行文件同级的 `languages/` 目录加载 UTF-8 JSON 语言包。开发环境也会探测仓库中的 `Ksword5.1/Ksword5.1/languages/`。语言包无需重新编译即可新增或更新，但应用启动后新增的文件需要重启才能进入语言列表。

## 文件格式

每种语言对应一个 `.json` 文件。文件名仅用于管理，语言身份由 `id` 决定：

```json
{
  "schema": "ksword-language-pack",
  "format_version": 1,
  "id": "en-US",
  "name": "English (United States)",
  "native_name": "English",
  "author": "KSword Team",
  "text_direction": "ltr",
  "fallback": "zh-CN",
  "translations": {
    "menu.settings": "Settings"
  },
  "source_translations": {
    "设置": "Settings"
  }
}
```

字段约束：

- `schema` 必须为 `ksword-language-pack`。
- `format_version` 当前必须为 `1`。
- `id` 使用 BCP 47 风格标识，例如 `en-US`、`zh-CN`、`ja-JP`。
- `name` 是英文名称，`native_name` 是该语言的本地名称。
- `text_direction` 可为 `ltr` 或 `rtl`，省略时为 `ltr`。
- `fallback` 可选，缺键时继续从指定语言包读取；循环引用会被截断。
- `translations` 的键和值都必须是字符串。键是稳定的语义 ID，不使用界面原文。
- `source_translations` 的键是现有源码中的界面原文，值是目标语言文本。它用于覆盖历史代码中的硬编码文案，并由审计工具保持完整。

单个语言包最大 32 MiB。格式错误、元数据不完整或包含非字符串翻译的包会被忽略。同一 `id` 出现多次时，仅采用搜索优先级最高的文件。

## 扩展流程

1. 复制 `languages/en-US.json` 并修改文件名和元数据。
2. 翻译 `translations` 和 `source_translations` 中的值，不修改键。
3. 对尚未翻译的键，可暂时删除并通过 `fallback` 使用其它语言。
4. 将文件放入 `Ksword5.1/Ksword5.1/languages/`，重新构建后会自动复制到输出目录。
5. 在设置的“界面语言”下拉框选择新语言并应用，检查主菜单、Dock 标题和设置页。

新增界面文本时，优先使用稳定语义键并同时在 `zh-CN.json` 和 `en-US.json` 增加同一键，再通过 `LanguageManager::bindText`、`bindToolTip`、`bindPlaceholder`、`bindTab`、`bindComboBoxItem` 或 `bindWindowTitle` 绑定控件。历史硬编码界面文本由运行时自动翻译层和 `source_translations` 兼容。代码内的 fallback 文案用于语言包缺失或损坏时保持界面可用，不能代替语言包条目。

语言包只允许对完整源字符串逐条翻译：运行时先按完整字符串精确匹配，再按原文中明确声明的 `%1`、`%2` 等占位符模板匹配。禁止把一个语言包条目的值作为另一个字符串的片段进行全局替换；这会把“简体中文”错误拼成“简体Medium文”。需要显示语言名称等元数据时，使用独立语义键，例如 `language.name.zh-CN`，不要把多个名称拼接后交给自动翻译。

## 完整性审计

仓库根目录的 `tools/i18n_language_pack.py` 会双向提取 C++ 和 UI 文件中的用户可见字符串，并校验中英文语言包是否缺项、是否存在陈旧条目、占位符和换行是否一致，以及目标语言中是否残留未允许的源语言文本。

```powershell
python tools\i18n_language_pack.py audit --source-root Ksword5.1\Ksword5.1 --zh-pack Ksword5.1\Ksword5.1\languages\zh-CN.json --en-pack Ksword5.1\Ksword5.1\languages\en-US.json
```

新增或修改界面文案后，先用 `sync` 同步双向源键，再用 `translate --model qwen3.5:4b` 分别补全英文和简体中文，最后运行 `audit`。Release 构建默认在编译前执行同一审计；只有临时诊断时才可通过 `/p:KswordSkipI18nAudit=true` 跳过，提交前不得跳过。
