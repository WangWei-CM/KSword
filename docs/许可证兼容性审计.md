# Ksword 许可证兼容性审计

审计日期：2026-07-16

本文记录仓库和当前发行流程的许可证边界，不替代许可证正文，也不构成法律意见。
根目录 `LICENSE` 只能授权项目版权所有者有权授权的材料，不能覆盖或改写第三方
许可证。

## 当前许可决定

项目方确认，本次切换所涉及的全部现有贡献者均同意将其贡献按 GNU General Public
License version 3 only（`GPL-3.0-only`）发布。当前许可结构为：

- `LICENSE`：未经修改的 GNU GPL version 3 完整正文；
- `PROJECT_LICENSE.md`：项目版权、适用范围和 `GPL-3.0-only` 声明；
- `COMMUNITY_COVENANT.md`：官方社区参与、项目身份和共同价值标准，不是软件
  许可证，也不附加于 GPL 权利；
- `CONTRIBUTING.md`：未来贡献按 `GPL-3.0-only` 入站，并适用 GPLv3 第 11 节
  的贡献者专利许可；
- `THIRD_PARTY_NOTICES.md` 与各组件 LICENSE/NOTICE：第三方权利链。

实际发布者仍应保存全部现有贡献者同意变更的可审计书面记录。本审计记录项目方
提供的确认，不代替这些原始证据。

## EASY-HWID-SPOOFER 结论

提交 `fb39a692`（`Integrate HWID dispatch controls`）同时加入了驱动 HWID
dispatch hook、UI/R3/R0 协议、构建项、GPLv3 正文和上游来源说明。此前自定义根
许可证包含禁止收费、托管限制等 GPLv3 不允许的附加限制，无法与可能形成一个
组合程序的 GPL 实现共同发布。

现在 `KswordARK.sys` 的项目自有部分也采用 `GPL-3.0-only`，因此这个许可证
不兼容已经关闭。关闭不等于材料义务完成：发行时仍必须准确记录使用的上游版本、
保留版权和修改声明，并提供能够生成所发驱动的完整对应源码、工程文件、配置和
构建脚本。逐行差异不能替代来源及权利链记录。

## Qt Charts 6.9.3 结论

仓库的 `.deps/Qt/6.9.3/msvc2022_64/sbom/qtcharts-6.9.3.spdx` 声明：

```text
PackageLicenseConcluded: LicenseRef-Qt-Commercial OR GPL-3.0-only
PackageLicenseDeclared: LicenseRef-Qt-Commercial OR GPL-3.0-only
```

源码、工程和当前二进制证明 `Ksword5.1.exe` 与 `KswordHUD.exe` 使用
`Qt6Charts.dll`。项目现在明确选择 Qt Charts 的 GPL-3.0-only 开源路径，而不是
依赖未证明的商业授权。主程序和 HUD 的项目自有部分同样按 `GPL-3.0-only`
发布，因此此前与自定义许可证的冲突已经关闭。

发行义务仍未自动完成。每个发行版必须包含或提供与实际 DLL 精确对应的 Qt 6.9.3
源码，保留 Qt 的版权、许可证及第三方通知，保存并随包提供适用的 SBOM/模块清单，
并确保项目完整对应源码能够构建该二进制。其他 Qt 模块若选 LGPL 路径，还必须
满足对应源码、替换和重新链接等 LGPL 条件。

## 社区公约能够保留和不能保留的诉求

`COMMUNITY_COVENANT.md` 可以作为官方社区的参与标准，要求透明说明修改、区分
官方与独立发行、负责任使用安全功能、诚实描述收费服务，并允许维护者管理官方
仓库、合作、徽章和认可。GPL 不授予暗示官方身份的商标权，项目可以在适用法律
范围内处理容易造成来源混淆的品牌使用。

社区公约不能成为复制、修改、托管、销售或再分发软件的条件，也不能因不遵守公约
而终止 GPL 权利。原自定义许可证中的禁止副本收费、禁止托管、禁止商业再分发和
必须另行书面授权等要求，不能以“社区公约”的名称继续作为版权限制。

## 历史版本

许可证变更只覆盖版权所有者有权变更且已同意切换的材料。项目方确认当前切换涉及
的现有贡献者均已同意，但重新发布旧 tag 时仍应核对该 tag 中每一项外部贡献和
第三方材料是否包含在同意范围内。不得仅凭当前根许可证推定一个历史包已经满足
GPL 对完整对应源码和通知材料的要求。

## 对外发行前仍须完成

两个主要“许可证互不兼容”问题已经通过选择 GPL-3.0-only 路径关闭；当前剩余的是
发行合规和证据问题。对外发行前至少必须完成：

1. 为实际发布的每个 exe、DLL、sys 和工具固定可复现的项目源码提交；
2. 提供 GPLv3 第 6 节要求的完整对应源码，包括项目源码、修改后的第三方源码以及
   控制编译和安装所需的脚本、工程和配置；
3. 固定 EASY-HWID-SPOOFER、Qt 6.9.3、Qt Advanced Docking System、FLTK 和其他
   预编译依赖的准确版本、上游来源、修改记录和源码取得方式；
4. 随包携带 `LICENSE`、`PROJECT_LICENSE.md`、`COMMUNITY_COVENANT.md`、
   `THIRD_PARTY_NOTICES.md`、组件级 LICENSE/NOTICE、Qt SBOM 和源码说明；
5. 对最终发行目录执行二进制导入和依赖清单核验，不能仅根据源码或旧包推测；
6. 确认没有来源不明、禁止再分发或与 GPLv3 不兼容的资源、SDK、库、工具、字体、
   图标和其他材料；
7. 按 `AGENTS.md` 的发行闸门验证压缩包内容，并保存本次审计证据。

在这些材料完成前，可以生成本地测试包，但不能标记为“GPL 发行合规已完成”或
“许可证审计通过”。
